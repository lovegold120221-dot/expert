# Local Video Generation

Locally hosted video generation on **macOS Apple Silicon** using **LTX-Video** for image-to-video animation.

## Pipeline: Still Image → Animated Video

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  Generate    │    │  LTX-Video  │    │  FFmpeg     │
│  Still Image │───▶│  Animation  │───▶│  Polish     │
│  (FLUX/MFLUX)│    │  (I2V)      │    │  (trim,      │
└─────────────┘    └─────────────┘    │  crop, fps)   │
                                       └─────────────┘
```

## Default Engine: LTX-Video

LTX-Video is the preferred engine for Apple Silicon because it:
- Runs natively on MPS (Metal Performance Shaders)
- Efficient memory usage (~8-10GB VRAM)
- Fast generation (30-60s for a few seconds of video)
- Good image-to-video quality
- Supports 24fps output at 768×512 or 512×768

### Image-to-Video Generation

```bash
# Generate video from a still image using LTX-Video
python3 -c "
import torch
from diffusers import LTXPipeline
from PIL import Image

# Load pipeline
pipe = LTXPipeline.from_pretrained(
    'Lightricks/LTX-Video',
    torch_dtype=torch.bfloat16,
    variant='bf16'
)
pipe = pipe.to('mps')

# Load source image (generated by local-image-gen)
source_image = Image.open('out/images/tech-office-bg.png')

# Generate video
video_frames = pipe(
    image=source_image,
    prompt='Slow subtle camera pan across the room, gentle motion, cinematic',
    negative_prompt='Fast movement, shaky, blurry, low quality',
    num_frames=49,           # ~2 seconds at 24fps
    num_inference_steps=50,
    guidance_scale=4.0,
    height=512,
    width=768,
).frames[0]

# Save as video file
import numpy as np
import cv2
fourcc = cv2.VideoWriter_fourcc(*'mp4v')
out = cv2.VideoWriter('out/videos/office-pan.mp4', fourcc, 24, (768, 512))

for frame in video_frames:
    frame_np = np.array(frame)
    frame_bgr = cv2.cvtColor(frame_np, cv2.COLOR_RGB2BGR)
    out.write(frame_bgr)

out.release()
"
```

## Video Generation Parameters

| Parameter | LTX-Video (I2V) | Notes |
|-----------|-----------------|-------|
| Resolution | 768×512 or 512×768 | Fixed model resolution |
| Frames | 49 (default) | ~2 seconds at 24fps |
| FPS | 24 | Standard cinematic frame rate |
| Inference steps | 30-50 | More = better quality, slower |
| Guidance scale | 3.0-5.0 | Higher = more prompt adherence |
| Duration | ~2-3 seconds per clip | Short clips, not long videos |

## Prompt Engineering for Video Animation

For LTX-Video image-to-video, the prompt describes the **motion** to apply to the still image:

```bash
# Gentle camera movement
prompt='Slow cinematic pan from left to right, gentle camera movement, steady'

# Subtle ambient motion
prompt='Gentle ambient motion, leaves swaying, soft natural movement, cinematic'

# Zoom effect
prompt='Slow zoom in, dramatic reveal, smooth transition, cinematic quality'

# No motion (static shot — use for talking head / interview style)
prompt='Static shot, no movement, steady camera'
```

### Negative Prompts

```bash
negative_prompt='Fast movement, shaky camera, blurry, low quality, distorted faces, unnatural motion, flickering, artifacts, jittery'
```

## Post-Processing with FFmpeg

After LTX-Video generates the raw clip, use FFmpeg to polish:

```bash
# Trim to exact duration
ffmpeg -i out/videos/raw-clip.mp4 -t 3 -c copy out/videos/clip-trimmed.mp4

# Change framerate
ffmpeg -i out/videos/raw-clip.mp4 -vf "fps=30" out/videos/clip-30fps.mp4

# Upscale for 1080p
ffmpeg -i out/videos/raw-clip.mp4 -vf "scale=1920:1080:flags=lanczos" out/videos/clip-1080p.mp4

# Slow motion (interpolate frames)
ffmpeg -i out/videos/raw-clip.mp4 -vf "setpts=2.0*PTS" -an out/videos/clip-slowmo.mp4

# Loop for longer duration
ffmpeg -stream_loop 3 -i out/videos/clip.mp4 -c copy out/videos/clip-looped.mp4
```

## Limitations

| Aspect | LTX-Video | Notes |
|--------|-----------|-------|
| Max duration | ~2-3 seconds per clip | Use looping or concatenation for longer |
| Resolution | 768×512 fixed | Must upscale for 1080p output |
| Motion complexity | Subtle/basic | No complex character animation |
| Faces | May have artifacts | Avoid close-ups of faces |
| Text rendering | Not supported | Cannot generate readable text |
| Audio | None | Add audio separately in video-editing-assembly |

## Fallback Chain

If LTX-Video cannot generate a suitable clip:

1. **Regenerate** the source image with a more static/compositional prompt
2. **Use FFmpeg** to create a Ken Burns effect from a still image (zoom/pan)
3. **Use ComfyUI** with a different workflow or video model

### FFmpeg Ken Burns Fallback

```bash
# Create pseudo-video from still image with Ken Burns zoom+pan
ffmpeg -loop 1 -i out/images/scene.png \
  -vf "zoompan=z='if(eq(on,1),1.0,min(zoom+0.005,1.3))':x='iw/2-(iw/zoom/2)':y='ih/2-(ih/zoom/2)':d=150:s=1920x1080:fps=30" \
  -c:v libx264 -crf 18 -t 5 -pix_fmt yuv420p out/videos/ken-burns-scene.mp4

# Pan across a large image
ffmpeg -loop 1 -i out/images/wide-scene.png \
  -vf "crop=1920:1080:mod(n-1\,200):0" \
  -c:v libx264 -crf 18 -t 6 out/videos/pan-scene.mp4
```
